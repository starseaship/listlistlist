# AI 解析阅读版式改版需求

## 背景

当前错题详情弹窗中的 AI 解析以一个长段落展示。内容完整，但用户在复习时需要快速抓住答案、考点、接续、错因和资料来源，长段落会增加阅读负担。

## 目标

把 AI 解析从“整段文字”改为“先结论、后细节”的阅读结构，让用户能在 5 秒内看到正确答案和核心语法点，在需要时再阅读原因和资料来源。

## 展示结构

1. 答案摘要
   - 显示正确答案选项和文本。
   - 显示我的答案。
   - 显示完整句子和中文意思。

2. 核心考点
   - 显示语法点，例如「〜にもかかわらず」。
   - 显示该语法的核心含义。

3. 接续
   - 从解析文本中提取“接续：”后的内容。
   - 使用独立卡片展示，避免混在长段落中。

4. 为什么这样选
   - 展示前后句的逻辑关系，例如“下雨”和“比赛仍举行”之间是逆接让步。

5. 错因提醒
   - 展示用户错选项和原因，例如「けれども」虽然也表示转折，但本题更偏固定句型。

6. 资料来源
   - 展示 Excel 索引定位信息。
   - 用较小字号展示，作为复习溯源信息。

## 交互要求

- 弹窗仍然保留原来的“完整选项”和“关闭”按钮。
- AI 解析不再默认展示为一整段。
- 如果解析文本不符合可解析格式，前端应回退为普通解释卡片，不能空白。
- 保持移动端可读：卡片纵向排列，文字不溢出。
- 保持桌面端可扫读：答案摘要可使用两列布局，其他内容使用清晰的卡片分区。

## 非目标

- 本次不修改数据库表结构。
- 本次不重写主应用路由。
- 本次不改变题目录入流程。
- 本次不改变 Supabase 查询权限。

## 验收标准

- 打开 Supabase 最新错题详情时，AI 解析显示为多个分区，而不是一个长段落。
- 正确答案、我的答案、完整句子、意思、接续、错因、来源可以单独定位。
- 没有解析文本时显示“暂无解析”。
- PWA 缓存版本更新，线上用户能拿到新版前端资源。
